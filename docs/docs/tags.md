# Tags

This page lists tags used across the documentation and what they represent.

## Business Documentation Tags

- **actions**: Documentation about user-facing actions and end-to-end operations.
- **todo**: Documentation related to TODO items (create/read/update/delete/toggle).
- **complete_day**: Documentation related to day completion (day closure / locking).

## Technical Documentation Tags

### Component Tags

- **component**: General React component documentation
- **react-component**: React-specific component implementation
- **ui**: User interface components
- **common**: Reusable common components (Button, Modal, etc.)
- **layout**: Application layout and structure components
- **calendar**: Calendar view components
- **timeline**: Timeline view components
- **todo**: Todo-related components

### Technical Category Tags

- **architecture**: System architecture and design patterns
- **state**: State management documentation
- **store**: Zustand store implementation
- **data-model**: TypeScript interfaces and data structures
- **utility**: Utility functions and helpers
- **hook**: Custom React hooks
- **styling**: CSS and Tailwind styling documentation
- **build**: Build configuration and deployment
